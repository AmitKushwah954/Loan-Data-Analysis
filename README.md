📊 Credit Risk Analysis using SQL

📌 Project Overview

This project focuses on analyzing a loan portfolio to identify credit risk patterns, default behavior, and high-risk borrower segments using SQL.

The analysis is designed to simulate real-world credit risk assessment in banking/fintech, helping in better decision-making for lending and risk management.

🎯 Objectives

Calculate overall default rate and portfolio exposure
Identify high-risk loan segments
Analyze borrower behavior based on income, DTI, credit history, and utilization
Build risk segmentation logic
Derive insights for risk-based decision making

🧠 Key Analysis Areas

1. Portfolio Overview
Total loans & total exposure
Default rate calculation
Loan term distribution
Interest rate vs default analysis

2. Risk by Loan Characteristics
Default rate by loan grade & sub-grade
Risk by loan purpose
Interest rate bucket analysis

3. Borrower Profile Risk
Risk by income segments
Impact of employment length
Verification status analysis
Home ownership vs default

4. Credit Behavior Analysis
Credit history vs default
Open accounts impact
Inquiry trends
Past delinquencies impact

5. Utilization & Exposure Risk
Revolving utilization vs default
High credit utilization impact
Default exposure calculation

6. Risk Segmentation
Created custom High / Medium / Low risk categories
Based on:
Income
DTI
Credit utilization

⚙️ SQL Concepts Used
Aggregations → COUNT(), SUM(), AVG()
Conditional Logic → CASE WHEN
Grouping → GROUP BY
Sorting → ORDER BY
Derived Metrics → Percentage calculations
Conditional Aggregation → SUM(CASE WHEN...)
